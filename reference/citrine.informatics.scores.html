Evaluates the expected magnitude of improvement beyond baselines for a given objective.

Parameters

  • objectives (list[Objective]) – objectives (e.g., maximize, minimize, tune, etc.) EIScore does not support more than 1 objective at this time.

  • baselines (list[float]) – best-so-far values for the various objectives (there must be one for each objective)

  • constraints (list[Constraint]) – constraints limiting the allowed values that material instances can have

class citrine.informatics.scores.LIScore(*, name: Optional[str] = None, description: Optional[str] = None, objectives: List[citrine.informatics.objectives.Objective], baselines: List[float], constraints: Optional[List[citrine.informatics.constraints.constraint.Constraint]] = None)¶

Bases: citrine._serialization.serializable.Serializable[LIScore], citrine.informatics.scores.Score

Evaluates the likelihood of scoring better than some baselines for given objectives.

Parameters

  • objectives (list[Objective]) – objectives (e.g., maximize, minimize, tune, etc.) If multiple objectives are specified they are evaluated independently, and ranked by the highest likelihood of exceeding a baseline. This should therefore not be used to simultaneously optimize multiple objectives.

  • baselines (list[float]) – best-so-far values for the various objectives (there must be one for each objective)

  • constraints (list[Constraint]) – constraints limiting the allowed values that material instances can have
